## 0.2.3: Web dashboard native parity

All platforms get a redesigned local dashboard at http://127.0.0.1:11400/ui/ that mirrors the macOS Settings window and menubar essentials.

### Features

- **Sidebar navigation:** Status, Backends, Models, Peers, and Config tabs (Agent, Discovery, Swarm, Routing, UI) plus Doctor & Tools
- **Menubar parity:** Discover, Copy client env, Status JSON, oMLX Admin, Refresh, and Save in the top action bar
- **Live routing stats:** Backends, peers, and models with 5s auto-refresh on the Status tab
- **Config editing in browser:** Save settings via `POST /netllm/v1/admin/config` (loopback-gated); restart banner when listen/port changes
- **LAN peer scan:** `POST /netllm/v1/admin/peers-scan` from the Peers tab (same as `netllm peers --subnet-scan`)
- **Theme-aware branding:** Light/dark logos match the macOS app; improved dark-mode contrast for code chips and config inputs

### Fixes

- **macOS menubar:** Agent status no longer shows "running" when another `netllm serve` owns the port but the menubar child process is dead (0.2.2.1 behavior, included in rebuilt macOS app)

### Install

- **All platforms:** `netllm serve` then open http://127.0.0.1:11400/ui/
- **macOS:** `llm-swarm-router.dmg` from [GitHub Releases](https://github.com/matthewdcage/llm-swarm-router/releases/tag/v0.2.3)
- **Linux:** `netllm_0.2.3_amd64.deb` or `netllm-0.2.3-*.rpm` (**alpha**)
- **Windows:** `netllm-0.2.3-windows-x64.zip` (**alpha**)
