## 0.2.3.4: macOS agent lifecycle + release gates

Patch release fixing menubar quit/orphan agent bugs on port 11400, hardening doctor and bundled Python probes, and gating macOS CI/releases on supervisor lifecycle tests.

### Fixes

- **macOS quit:** `applicationWillTerminate` no longer deadlocks on MainActor while stopping the agent; quit releases `:11400` even when the supervisor is `failed` or `stopped` with an orphan listener
- **Menubar start:** bundled `serve` uses `--replace` to avoid exit-code-0 “already running” loops and false “Auto-restart failed” states
- **Doctor:** when `NETLLM_SUPERVISED=menubar`, warns if the menubar supervisor reports the agent is not running while the port is still held
- **Bundled Python:** loopback provider scans use `verify=False` so missing CA bundles in the venvstacks embed do not break `/netllm/v1/status`
- **Updates:** GitHub releases without a macOS DMG asset surface “download manually” instead of failing silently

### CI / QA

- **`scripts/test-menubar-lifecycle.sh`:** quit cleanup, control-socket stop/restart, and `--replace` scenarios (L1–L5)
- **GitHub Actions:** `menubar-lifecycle` job on `macos-14`; release `build-macos` runs e2e + lifecycle before DMG packaging
- **Docs:** [macOS release readiness gates](../solutions/macos-release-readiness.md)

### Upgrade notes

- **From 0.2.3 (no update UI):** install this DMG once manually ([v0.2.3.2](v0.2.3.2.md)+ adds in-app updates). After that, menubar **Updates** works.
- **Stuck agent on :11400:** quit the menubar app (this build cleans up on quit) or run `lsof -ti :11400 | xargs kill -9` before relaunch.
