## 0.2.3.5: macOS update UX + agent port recovery

Patch release fixing stuck in-app updates, blank Settings on launch, and the menubar supervisor landing in permanent **Failed — Agent exited with code 0. Auto-restart failed.** while an orphan agent still serves on `:11400`.

### Fixes

- **Auto-update:** resume verified cached DMGs on launch; download progress in menubar and Settings; install confirmation activates the app; notifications and `~/Library/Application Support/netllm/logs/update.log`
- **Settings window:** remove empty SwiftUI `Settings { EmptyView() }` shell that opened a blank **llm-swarm-router Settings** window on DMG install
- **Agent supervisor:** `releaseListenPort()` before manual start and auto-restart; adopt healthy listeners on the listen port instead of failing; background port monitoring self-heals `.failed` / `.stopped` when `/health` still responds
- **Launch:** reconcile an already-listening agent before spawning a new supervised child

### Upgrade notes

- **From 0.2.3.2+:** menubar **Updates → Install** (or web dashboard `/ui/`). If stuck on an old build, install this DMG once manually.
- **Stuck “Failed / Agent not listening” with backends still online:** quit and relaunch the menubar app (this build adopts or clears the orphan within seconds), or click **Start** in Settings.
