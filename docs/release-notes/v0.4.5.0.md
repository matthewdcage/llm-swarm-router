# 0.4.5.0: oMLX-style telemetry — Serving stats, System Stats, unified API

Feature release bringing oMLX-inspired observability to the mesh router without turning netllm into an inference server. Router-owned metrics (requests, tokens, capacity) are unified with optional oMLX admin stats in one API, surfaced on the web dashboard and in a native macOS menubar menu.

## New

- **`GET /netllm/v1/telemetry`** — Unified telemetry for dashboard and menubar (`scopes=router,omlx`, `watch=1` for live oMLX probes, `history=60` for sparkline buffers). Router session and all-time counters persist to `~/.config/netllm/stats.json`. Schema: [telemetry-api.md](../telemetry-api.md).
- **Web dashboard Serving tab** — Session and all-time router stats, optional oMLX PP/TG and cache panels when a local oMLX 0.5.2+ admin API is reachable, sparklines, and enriched Status fields (routed requests, capacity rejections, peer warnings).
- **macOS menubar NSMenu** — AppKit `MenubarController` replaces the popover-primary UX: **System Stats** fly-out (CPU E/P, GPU, memory sparklines via native `HostSampler`), **Serving Stats** submenu (router + oMLX session/all-time rows), **Models** submenu with favorites, optional menu-bar gauge items (CPU/GPU/MEM/LIV) under Settings → Appearance.
- **`ui.model_favorites` and menubar appearance toggles** — Star models in the dashboard; favorites filter in Models menu; config export/import round-trips new `ui.*` fields.
- **Prometheus token counters** — `netllm_prompt_tokens_total` and `netllm_completion_tokens_total` on `/metrics`.

## Fixed

- **Status tab host summary** — Dashboard loads a `watch=0` telemetry snapshot on refresh so host CPU/memory rows populate without opening Serving first.
- **Doctor open-LAN endpoint test stability** — Endpoint doctor checks no longer fail on machines with auth-gated local backends unrelated to LAN notes.

## Upgrade

macOS (menubar, recommended on macOS 26+):

```bash
git clone https://github.com/matthewdcage/llm-swarm-router.git
cd llm-swarm-router && git checkout v0.4.5.0
uv sync && uv pip install venvstacks
apps/netllm-mac/Scripts/build.sh release
packaging/scripts/macos-app-install.sh --source apps/netllm-mac/build/Stage/llm-swarm-router.app
```

When a notarized DMG is available from [GitHub Releases](https://github.com/matthewdcage/llm-swarm-router/releases):

```bash
./scripts/upgrade-mac-app.sh ~/Downloads/llm-swarm-router.dmg
```

**CLI only:** `uv sync && ./netllm init && ./netllm serve` — dashboard at http://127.0.0.1:11400/ui/ (open the **Serving** tab for telemetry panels).

Verify telemetry:

```bash
curl -sf 'http://127.0.0.1:11400/netllm/v1/telemetry?watch=0' | python3 -m json.tool
```
