## 0.3.0.0: Apple AI Phase 1 & 2

First Apple-platform milestone: router bridge parity, configurable routing policies, and native macOS integration (App Intents, MenuBarExtra, design tokens, login item).

### Phase 1 — foundation

- Anthropic streaming `tool_use` for local editor workloads
- OpenAI cloud failover parity with Anthropic (real API keys only)
- `X-Netllm-Local-Only` header blocks cloud inject
- Dynamic oMLX admin URLs from agent status
- Keychain storage for cloud API keys

### Phase 2 — platform

- **App Intents + Shortcuts** — list models, agent status, start/stop, copy client env, open dashboard
- **MenuBarExtra** SwiftUI menubar with popover (replaces legacy `NSStatusItem`)
- **Design tokens** shared between macOS app and web dashboard (`dashboard-tokens.css`)
- **Launch at login** via `SMAppService`
- **`[[routing.policies]]`** — model-prefix rules, `allow_cloud`, `prefer_provider`
- **Liquid Glass** card chrome when built with macOS 26 SDK (fallback chrome otherwise)
- VoiceOver labels in Settings and menubar popover

### Install / upgrade

- **macOS:** GitHub Release DMG or menubar **Updates** after publish
- **Source:** `git pull && uv sync`
- **Linux/Windows:** deb/rpm/zip from the same release tag
